Examples and documentation

Width and height

An element's width and height can depend on several factors, such as the window's size, the parent element's size and/or the element's content and css properties. While css media queries are limited to the window's width and height, EQ targets the width of the element itself.

Width

.width:max-width(20) { background-color: gray; } .width:min-width(40) { background-color: #fb9c7f; } .width:min-width(60) { background-color: black; } .width { display: inline-block; height: 40px; }

Height

.height:max-height(20) { background-color: gray; } .height:min-height(40) { background-color: #fb9c7f; } .height:min-height(60) { background-color: black; } .height { display: inline-block; width: 40px; }

Text length

Query elements by the number of characters they contain.

.text-length:max-text-length(10) { color: #fb9c7f; } .text-length:min-text-length(15):max-text-length(20) { color: green; }

What about performance?

Use it sparingly

EQ uses JavaScript for rendering, and is therefore not nearly as fast as native CSS. Use it sparingly and do not apply it to a large number of elements. For example, instead of applying an element query to a large number of <li> elements, apply it to the parent <ul> instead (if possible):

Wrong
li:max-width(800) { display: block }
Right
ul:max-width(800) li { display: block }

Apply unambiguous styles

What will happen in this scenario?

.my-selector { width: 400px }
.my-selector:min-width(100):max-width(800) { width: 0 }
.my-selector:min-width(0) { width: 400px }

EQ will prevent infinite recursion by not allowing an endless, synchronous loop to occur.
However, not doing contradictory things in the first place will improve performance.

Use native CSS where propriate

The purpose of EQ is to enhance native CSS, not to replace it.
Use media queries and other techniques as usual, then add EQ to make things more awesome.
